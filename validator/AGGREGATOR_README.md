# G1 Public Key Aggregator Contract

## 📋 概述

`G1PublicKeyAggregator.sol` 合约实现了使用 EIP-2537 G1 加法预编译指令聚合多个 BLS12-381 G1 公钥的功能。

## 🎯 功能特性

### 核心功能
- ✅ **单次聚合**: `aggregatePublicKeys(bytes[] calldata publicKeys)` - 将多个 G1 公钥聚合为一个
- ✅ **聚合验证**: `aggregateAndVerify()` - 聚合公钥并与预期结果比较
- ✅ **批量聚合**: `batchAggregatePublicKeys()` - 批量处理多组公钥聚合

### 验证功能
- ✅ **公钥验证**: 验证输入公钥格式正确性
- ✅ **长度检查**: 确保所有公钥都是 128 字节 EIP-2537 格式
- ✅ **无限点检测**: 识别和处理点 at infinity

### 工具功能
- ✅ **Gas 估算**: 预估聚合操作的 Gas 消耗
- ✅ **预编译地址**: 返回使用的 G1 加法预编译地址 (0x0b)

## 🔧 技术实现

### 预编译调用
合约使用 EIP-2537 G1 加法预编译指令 (地址 0x0b) 进行点加法运算：

```solidity
// 输入：256 字节 (两个 128 字节的 G1 点)
// 输出：128 字节 (一个聚合的 G1 点)
(bool success, bytes memory output) = G1_ADD_PRECOMPILE.staticcall{
    gas: PRECOMPILE_GAS
}(additionInput);
```

### 数据格式
- **输入格式**: EIP-2537 标准 128 字节 G1 点编码
- **输出格式**: 同样的 128 字节 G1 点编码
- **批量输入**: `bytes[]` 数组，每个元素 128 字节

## 🚫 限制和注意事项

### 网络支持
- **问题**: 大多数现有网络（包括 Sepolia）尚未激活 EIP-2537
- **影响**: G1 加法预编译指令 (0x0b) 不可用或返回空结果
- **解决方案**: 需要使用支持 EIP-2537 的网络或等待网络升级

### 测试环境
- **Forge/Anvil**: Mock 预编译实现复杂，测试有限
- **建议**: 使用支持 EIP-2537 的专用测试网络
